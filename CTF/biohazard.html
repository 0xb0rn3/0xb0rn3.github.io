Context

Overview

Biohazard is a Resident Evil-themed CTF that simulates exploring the Spencer Mansion. Rather than a single vulnerability, it chains together a long sequence of skills: web room enumeration with hidden HTML comments, form-based item collection, multi-step encoding/decoding puzzles across base64/base32/base58, FTP steganography extraction across three JPEG images, GPG decryption, and two layers of Vigenere cipher to finally pivot into SSH and escalate to root via a sudo-enabled user.

Every collectible item found in the mansion is required to unlock the next area — skip one and the chain breaks. The full path from HTTP root to root shell requires 10 sequential phases.

ATTACK CHAIN
Phase 1  Web Rooms    → emblem, lock_pick, blue_jewel (HTML source mining)
Phase 2  Item Collect → Crests 1-4 (form POSTs, ROT13, base32, bar room chain)
Phase 3  Decode Chain → 4x multi-encoding → FTP user:pass
Phase 4  FTP + Stego  → 3 JPEG key parts (steghide / EXIF / ZIP) → GPG pass
Phase 5  GPG Decrypt  → helmet_key → unlock study room + hidden closet
Phase 6  SSH Access   → umbrella_guest (T_virus_rules)
Phase 7  Vigenere x2  → weasker:stars_members_are_my_guinea_pig → sudo root

Phase 4 — FTP Files & Three-Part Key Assembly

FTP as hunter reveals five files: three JPEG images, an encrypted GPG file, and a note from Barry hinting that the helmet key is in the GPG file and a /hidden_closet/ path exists. Each JPEG hides a key fragment using a different steganography method:

BASH

    "ftp://TARGET/{001-key.jpg,002-key.jpg,003-key.jpg,helmet_key.txt.gpg}" \
    -o "#1" # download all four files

# 001-key.jpg → steghide (empty password)
$ steghide extract -sf 001-key.jpg -p ""

# 002-key.jpg → EXIF Comment field
$ exiftool 002-key.jpg | grep Comment

$ cp 003-key.jpg 003-key.zip && unzip 003-key.zip

# Assemble + base64 decode

Defense

Takeaways

Never embed paths in HTML comments
Comments like <!-- href="/mansionmain/" --> hand attackers the sitemap. Server-side routing should never leak internal paths client-side.
Redirect headers reveal secret URLs
Secret room URLs returned in Location headers are trivially observable with curl -v. Security through obscurity is not access control.
Symmetric ciphers ≠ encryption
Vigenere and ROT13 are encodings. Once the key is found anywhere in the system, all ciphertexts are broken. Use AES-256-GCM for secrets.
Steganography is not security
Data hidden in EXIF fields or appended ZIP archives is trivially recovered with exiftool and unzip. Credentials must never be hidden in media files.
Audit sudo group membership
Any user in the sudo group with a guessable or derivable password is a one-command root. Review /etc/sudoers and apply least privilege.
Sequential dependency chains
Each item was required for the next phase — perfect for CTF design, but also a reminder that credential reuse across services creates exploitable pivot chains in real environments.
Automation

Full-Chain Exploit Script
